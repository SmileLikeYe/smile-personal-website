# Prototype Instructions

Run the local server yourself and open the preview in the in-app browser. Do not give the user server-start instructions when you can run it.

## Local Private Context

If `AGENTS.local.md` exists at the repository root, read it before making project-specific decisions. Treat it as local private context for deployment notes, personal logs, and machine-specific workflow preferences.

Never stage, commit, or publish `AGENTS.local.md`.

Before making substantial visual changes, use the Product Design plugin's `get-context` skill when the visual source is unclear or no longer matches the current goal. When the user gives durable prototype-specific design feedback, preferences, or decisions, record them in `AGENTS.md`.

When implementing from a selected generated mock, treat that image as the source of truth for layout, component anatomy, density, spacing, color, typography, visible content, and hierarchy.

## Prototype Design Decisions

- Homepage concept (2026-07 改版): 个人积累与分发平台 — 宣传自己 + 沉淀影响力。Narrative loop: 写 WRITE（文章）→ 建 BUILD（产品/项目）→ 淀 DISTILL（skill）。内容先在本站沉淀，再分发到 GitHub / 小红书等平台（About 区的"全平台信号台"跟踪各渠道 LIVE/SOON 状态）。
- Layout: editorial magazine bones (giant serif section wordmarks, numbered hairline lists, mono micro-labels) but the sections must NOT clone the sac-ai.com reference — the user flagged 雷同 as a risk (2026-07-07, citing github.com/Leonxlnx/taste-skill: own voice, no template patterns). The site's own identity thread is the "Smile Hu as a large model" metaphor, carried through every section: hero training panel → About "Serving Endpoints" table (GET path → 200/SOON status rows, NOT Sac's 信号台 with equalizer bars) → Writing = "Training Log" with ckpt-NN rows (NOT a feature-card magazine grid) → Skills = "Adapters" → Build full-width rows + "now training:" line (NO git-log/queued decorations) → asymmetric "Let's talk." two-column contact (NO marquee, NO hand-drawn underline, NO rail lines/corner crosses). When adding sections, extend the model metaphor rather than borrowing another site's signature components.
- Do not use `AI Native` as a visible hero label.
- Use the provided real portrait directly as an asset; do not ask ImageGen to redraw or imitate the face.
- The hero right side is the restored pre-redesign **training panel** (`.training-panel`, the "Smile Hu as a large model" collage): 01 Pre-training / model core (flow-background art with the portrait) / 02 Fine-tuning adapters / 03 Runtime Context / 04 Outputs / 05 Eval Loop, with the ambient glow/scan/wash/pulse animations. The user explicitly asked to keep this panel exactly as it was (2026-07-06) after a condensed "spec card" replacement was rejected — do NOT remove or minimize it in future redesigns. It collapses to a stacked card grid ≤1180px (photo circle appears there via `.model-photo`). The panel doubles as the homepage table of contents (2026-07-08): each numbered block's title is a `.panel-link` anchor — 01 Pre-training→#about, 02 Fine-tuning→#writing (Training Log), 03 Runtime Context→#context, 04 Outputs→#skills (Adapters), 05 Evaluation Loop→#build (…→Ship). Homepage section order and kicker numbers MUST stay aligned with this 01-06 mapping (Contact=06).
- All editable site copy (profile, about, channels, skills, projects, queued, marquee) lives in `src/content/site.js`; components read from it. Values marked `[MOCK]` await real data from the user.
- Skills live in `src/content/skills/*.md` with `kind: original | curated`. Curated skills (收藏) MUST keep `author` + `source` frontmatter and the attribution blockquote at the top — they are other people's work (李继刚, Emil Kowalski, Leonxlnx) presented as a curated library with 使用心得; never present them as the user's own. Home shows the first 5 (originals sort first); full searchable list in #library/skills.
- Context 收藏区（首页 05 / #library/context / #ctx/<slug>）：条目在 `src/content/context/*.md`。入库只走 `collect` skill 的流程（抓取 → LLM 总结带原文筋骨 → 必须有"我的批注" → 建议 internalized 去向）。收藏的终点是内化——不要生成没有批注的条目。
- Blog posts target interviewers at Chinese AI companies and cross-platform readers; new posts are written in Chinese by default. UI mixes English serif wordmarks + mono English micro-labels + Chinese body copy (editorial style).
- Post deep links use the `#post/<slug>` hash format; plain `#section` anchors are reserved for navigation. Articles render as a dedicated second-level page (`PostPage`, App-level hash routing with AnimatePresence): home unmounts, page scrolls to top, a fixed 2px Lake Blue reading-progress bar (motion `useScroll` → scaleX) sits above the header, and back (button or browser) returns to the home page anchored at #writing.
- Perpetual ambient animations (all CSS, disabled under prefers-reduced-motion): traveling dot on the endpoints 写→建→淀 pipe, typewriter status line in Contact (`TypeLine`, phrases in site.js `statusLines`), spinning `loop-ring` in the footer ("eval loop running"). Keep them tiny and mono-styled; hover rows slide their serif titles 6px right.
- Visual tokens still follow the Monad style reference (`docs/design-reference.md`, from styles.refero.design): warm parchment canvas #f6f3f1 (never pure white) with subtle dot-grid + paper-grain noise, serif headlines at weight 400 only (Newsreader + Chinese serif fallback), ALL UI strings in mono (IBM Plex Mono), single Lake Blue #2b59d1 accent, 1px ash hairline borders + pill buttons instead of shadows. Pastel washes were removed in the editorial redesign.
- The `motion` library is loaded via `LazyMotion features={domAnimation} strict` — always use `m.*` components (never `motion.*`, strict mode throws). Current motion uses: layoutId nav dot, staggered hero entrance, AnimatePresence writing grid↔reader transitions. Marquee/pulse loops stay in CSS.
- Animation rules (from Emil Kowalski's design-eng standards): UI transitions under 300ms, strong ease-out `cubic-bezier(0.23, 1, 0.32, 1)`, only animate transform/opacity, press feedback `scale(0.98)` on :active, stagger 30-80ms. Slow ambient washes are the one exception to the 300ms rule.
- Sections below the hero use the `[data-reveal]` + `.revealed` animation pattern (IntersectionObserver, `card-boot` keyframes). Note for QA scripts: full-page screenshots must freeze animations, and programmatic scrolling must use `behavior: "instant"` because the site sets `scroll-behavior: smooth`.
