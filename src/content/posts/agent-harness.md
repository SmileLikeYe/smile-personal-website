The useful metaphor is not "I use AI". The useful metaphor is: I am a product model with a harness around me.

My base training is computer science from Tongji University. The adapters are the systems I keep shipping: PIN AI, mobile local AI, MTEB evaluation work, Codex skills, and product workflows. The runtime context is what changes every day: user problems, codebases, notes, meetings, device constraints, and open-source feedback.

The harness has a simple shape:

| Layer | What it means in my work |
| --- | --- |
| Context | Real user needs, repo state, device logs, product constraints |
| Tools | Codex, GitHub, local scripts, browser QA, iOS/device loops |
| Skills | Repeatable playbooks that turn messy work into execution |
| Evaluation | Build checks, screenshots, PR review, benchmark contribution |
| Output | Shipped products, open-source PRs, notes, and reusable systems |

The goal is not to look like a model. The goal is to behave like one: take context, run the right skills, evaluate the result, then ship.
